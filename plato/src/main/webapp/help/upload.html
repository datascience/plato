How to correctly specify requirements using Freemind

This page explains the syntax that should be used when specifying objective trees with the mind-mapping software Freemind.

Tree structure

In general, just construct a mindmap containing the structure, save it and upload the .mm file.
The leaf nodes of the mindmap will be taken either

  • as leaf nodes, if you unchecked the box 'has units', or
  • as scale definitions for the level above which contains the leaf nodes, if you checked the box.

Using scales

If you choose to define scales, you have to specify a scale for every leaf node.
This means you have to add a child node to every leaf with a text specifying the type of scale. The following types are possible:

  • Yes/No scales can be specified by the text 'Y/N'.
  • If you want to specify arbitrary values such as 'same/slight differences/unusable', you should separate the values by a slash.
    For example, Availability of documentation in the provided example can have one of the values
    • public,
    • limited, or
    • none.
  • If you enter any other text, this is assumed to be the measurement unit of a numeric scale, such as 'seconds' or 'MB'.
