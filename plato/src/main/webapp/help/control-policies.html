How to specify control policies.

This page explains the semantic model that shall be used when specifying control policies.

What are control policies?

The current version of Plato supports a first, simple but effective way of policy-driven planning. A preservation plan can be based on a preservation scenario which links the content set that should be preserved to the user community it should be preserved for. Plato then, automatically, builds the requirements tree based on the control policies defined for this scenario.

Writing control policies

Control policies have the general form policy(measure, value, qualifier, modality)

  • measure A measure that the objective pertains to. Measures are standardised and currently curated by the Plato development team. Please contact us at plato@ifs.tuwien.ac.at to obtain a the current list.
  • value A value associated with the measure.
  • qualifier A qualifier (equals, greater than, less than, greater equal, lower equal). Equality by default.
  • modality A modality that describes whether the particular property-value pair is present or not. (MUST, SHOULD, SHOULD NOT, MUST NOT, MAY)

Example policy

<owl:NamedIndividual rdf:about="FormatDocumentationMustBeFreelyAvailable">
   <rdf:type rdf:resource="&control-policy;FormatObjective"/>
   <control-policy:value rdf:datatype="&xsd;string">yes-free</control-policy:value>
   <control-policy:property rdf:resource="&measures;147"/>
   <pw:organizationScope rdf:resource="austrian_state_archives"/>
   <control-policy:modality rdf:resource="&modalities;MUST"/>
</owl:NamedIndividual>
