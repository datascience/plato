Plan Settings

Private
If a plan is set to private (which is the default), only the creator can see it.
Publish report
For private plans, you can still make the uploaded report available to other users. They will not be able to change or open your plan, but see it in the list and download the report.
Uploading a report
  • Open your plan and go to "Validate plan" (or Analyse Results, if you haven't progressed so far yet).
  • Expand everything, i.e. all the nodes in the trees that you want to see, and the evaluation and transformation section.
  • Print to PDF with the following settings: orientation landscape, two pages on one. This gives you a very compact layout in portrait with two landscape pages in small font. The DEMO plans provide such examples.
  • Upload the resulting PDF file here.

Define Basis

Identification code
Institutional unique identifier for the preservation plan
Document Types
This is a a general description of the set of objects you are creating a preservation plan for.
Original technical environment
The original set of software and hardware that was used to create and render the objects. This description should be as detailed as possible to allow a solid comparison between environments and objects.
Plan Name
Please provide a name for the preservation plan.
Plan Description
This is a short description of the planning activity, basically a short summary.
Responsible Planner
Responsible person for the planning activity. Please provide full name, job title, and contact.
Organisation
Name of the organisation that is carrying out the planning activity.
Mandate
Mandate of the repository (see TRAC A1.1)
Planning Purpose
What is the purpose of the current planning activity?
Are you, for example, revising an existing plan because of changed environment factors or new objects?
Or are you defining a first plan for a set of objects that you have just received?
Designated community
Identification of the designated community or communities and their associated knowledge base(s) (TRAC A3.1)
Applying policies
Relevant polices for preservation planning activites (TRAC A3.3) - which policies are influencing your planning activity?
Relevant organisational procedures and workflows
Relevant procedures, rules, manuals, handbooks and workflows for preservation planning endeavours (TRAC A3.3)
Contracts and agreements specifying preservation rights
Relevant Contracts and deposit agreements of preservation rights (TRAC A5.2)
Reference to agreements of maintenance and access
Relevant contracts and deposit agreements specifying all appropriate aspects of maintenance and access of the collection. (e.g. submission agreements, deposit agreements) This is especially relevant for data which are owned by third parties. Agreements may cover restrictions or requirements for access and maintenance, e.g. provide DIP in specific format or repository is responsible for error fixing.
Relations
Are there any relations to other preservation plans? Is this plan for example based on another one replacing another plan, etc.?
Trigger
Events that triggered this preservation planning activity (eg. new objects in the collection, a periodic review of the plan, changes in environment or requirements).
The following types of triggers are foreseen:
New collectionA plan is created for a new set of objects.
Periodic ReviewA scheduled review has led to the decision that an existing plan shall be revised.
Changed EnvironmentContextual influence factors have changed that justify a revision of a plan.
Changed ObjectiveThe goals of the institution have changed and require revising an existing plan.
Changed Collection ProfileThe profile of a set of objects has changed in such a way that current measures are not sufficient to keep the objects alive.

Run Experiment

Run experiment description
Documentation about running the experiments for every sample object.
Report
This text area contains a summary of the reports aggregated over all sample objects.
For automated migration services, usually the commandline output of the migration tools is automatically aggregated and displayed here. Emulators will have different logging output.
If a preservation action service fails with an error, this will be recorded here together with available information on the type of error, if applicable and available.
Resulting Files
The resulting files after the experiments have been run. (eg. for migration the migrated files).
For migration web services that are executed by the planning tool, these files will be retrieved automatically. For all experiments that you are carrying out yourself, please upload all resulting files and documentation here.

Comments

Analyse Results

Recommendation and reasoning
The recommendation for one of the potential alternatives is based on an analysis of the strengths and weaknesses of the considered strategies. The reasons and observations should be recorded here to document the reasons for the decision. This is an essential part of the decision and should not be disregarded!
The recommendation forms the basis for specifying a preservation plan for the set of objects in the fourth phase of the planning process and thus needs to be thoroughly documented.

Define Preservation Plan

Currency
The currency used for the cost model figures.
CIG (Integrate new solution)
Costs of integration of the new solution. Contains costs for developing, software and hardware. Part of the Life-Cost-Model for digital preservation.
CPE (Perform PA)
Total costs of performing the Preservation Action over the whole collection. Part of the Life-Cost-Model for digital preservation.
CQA (Quality Assure PA)
Total costs of Quality Assurance for Preservation Action over the whole collection. Part of the Life-Cost-Model for digital preservation.
CRM (Record PA Metadata)
Total costs of recording metadata about the performed preservation action for the whole collection. Part of the Life-Cost-Model for digital preservation.
PA Total (CIG+CPE+CQA+CRM)
Total costs for the Preservation Action. This should be a sum of CIG, CPE, CQA and CRM. Part of the Life-Cost-Model for digital preservation.
Reingest
Total costs for the reingest of the objects on which the preservation action has been performed. Part of the Life-Cost-Model for digital preservation.
TCO
Total Cost of Ownership. If you use this model for calculating the costs of your preservation action and re-ingest, enter the amount here.
Remarks
Remarks about the figures. If you use a costmodel different to TCO or Life, you can enter the data here as well.
Responsible person for execution
Enter the person responsible for executing the preservation plan here.
Responsible person for monitoring
Enter the person responsible for approving the preservation plan here.
